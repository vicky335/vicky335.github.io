F2E面試題 - JavaScript 閉包(Closure)

Aug 1, 2023 #f2e #closure #javascript

JavaScript 中的闭包（Closure）是指一种函数和其相关的引用环境的组合。当一个函数可以访问其定义时所处的词法作用域中的变量，即使在函数被调用时，这些变量已经不再存在，那么这个函数就被称为闭包。

闭包的作用是可以访问外部函数的变量和参数，同时也可以访问全局变量。闭包可以用于许多场景，比如实现模块化、延迟执行、实现私有变量和方法、避免全局污染等。

在 JavaScript 中，当一个函数返回另一个函数时，后者可以访问前者的变量，即使前者已经执行完毕并且其执行上下文已经被销毁。这个后者就是一个闭包。

以下是一个使用闭包实现计数器的例子：

function createCounter() {

  let count = 0;

  return function() {

    count++;

    console.log(count);

  }

}



const counter = createCounter();

counter(); // 输出 1

counter(); // 输出 2

counter(); // 输出 3

在上面的例子中，createCounter 函数返回一个内部函数，这个内部函数访问了 createCounter 的作用域中的变量 count，并且每次被调用时都将 count 的值加 1 并输出。由于 counter 函数是 createCounter 的返回值，所以它是一个闭包，可以访问 createCounter 的作用域中的变量 count，并且每次执行时都能正确地记录 count 的值。

闭包的特性包括：

  1. 可以访问定义时所在的词法作用域。

  2. 可以在函数外部访问闭包内部的变量。

  3. 闭包中的变量会常驻内存，不会被垃圾回收机制回收，因此需要注意内存泄漏的问题。

总结来说，闭包是 JavaScript 中非常重要的概念，它可以用来创建模块、实现私有变量和方法、避免全局污染等。在使用闭包时，需要注意内存泄漏的问题，避免造成不必要的性能问题。
